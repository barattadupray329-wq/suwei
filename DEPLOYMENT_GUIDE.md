## 🔄 数据流向和同步机制

```
┌─────────────────┐
│  主电脑         │
│  租赁数据       │
└────────┬────────┘
         │ 推送
         ▼
┌─────────────────────────────────────┐
│      坚果云（云端存储）              │
│  625730448@qq.com                   │
└────────┬────────────────────────┬───┘
         │ 同步                   │ 同步
         ▼                        ▼
┌──────────────┐           ┌──────────────┐
│ 店员电脑 1   │           │ 店员电脑 2   │
│ 租赁数据     │           │ 租赁数据     │
└──────────────┘           └──────────────┘
```

### 同步特点
- **实时性**：文件编辑后立即同步
- **双向**：任何电脑修改都会同步到其他电脑
- **安全**：坚果云会保留版本历史，支持恢复
- **自动**：无需手动操作，后台自动同步

## 🐛 故障排除

### 问题 1: 启动时卡在"同步数据"

**原因**：坚果云未安装或未运行

**解决**：
1. 检查坚果云是否安装
2. 确保坚果云已启动（右下角托盘)
3. 检查网络连接
4. 重启坚果云

### 问题 2: 看不到其他电脑修改的数据

**原因**：数据同步延迟或文件冲突

**解决**：
1. 等待 30 秒让坚果云同步
2. 刷新应用数据（重新选择记录）
3. 重启应用
4. 检查坚果云同步状态

### 问题 3: 应用无法启动

**原因**：数据文件损坏或软链接失效

**解决**：
1. 检查租赁数据文件夹是否存在
2. 重新创建软链接（使用 mklink 命令）
3. 恢复数据库备份

### 问题 4: 更新失败

**原因**：网络中断或磁盘空间不足

**解决**：
1. 检查磁盘空间（至少 500MB）
2. 重启应用重试更新
3. 从备份恢复旧版本（`.version_backup` 文件夹)

## 📞 技术支持

### 常见命令

```powershell
# 查看同步状态
ls "租赁数据" | Measure-Object -Sum FileLength

# 重新创建软链接
cmd /c mklink /D data "D:\path\to\租赁数据"

# 清理更新缓存
Remove-Item ".update_cache" -Recurse -Force

# 查看应用日志
Get-Content "app.log"
```

### 日志文件

应用会生成日志文件（如果启用）：
- `app.log` - 应用日志
- `.version_backup/` - 版本备份
- `.update_cache/` - 更新缓存
