# 项目开发规则

## 版本划分

项目分为四个板块：

1. **核心逻辑** - 共享的业务逻辑和数据管理
2. **单机版** - 单台电脑使用的版本
3. **网络版** - 支持多端同步的网络版本
4. **局域网版** - 局域网同步版本（暂时不开发）

## 开发约定

1. **默认修改核心逻辑**：当用户指定修改某个版本（如单机版、网络版）时，实际上默认修改的是核心逻辑部分

2. **核心逻辑变更同步**：核心逻辑的任何变动都需要同步到其他版本（单机版、网络版），确保各版本核心逻辑保持一致

3. **版本差异**：各版本之间的差异仅在于：
   - 同步方式（本地存储 vs 网络同步）
   - 界面配置（可能略有不同）
   - 网络相关功能

4. **局域网版**：暂时不开发，保持为占位状态

## 当前版本状态

- 核心逻辑：✅ 开发中
- 单机版：✅ 可用
- 网络版：✅ 可用（支持 HTTP 同步服务器）
- 局域网版：⏸️ 暂停开发
