## 📈 技术实现详解

### 1. KpiCard 卡片组件 (Lines 13-227)

**UI元素**:
- 标题标签 (灰色, 10pt)
- 大号数值显示 (白色, 24pt粗体)
- 趋势箭头 (↑↓→, 颜色编码)
- 单位标签 (灰色, 10pt)
- 副标题 (灰色, 9pt)

**状态管理**:
- 正常: 深灰背景 (#2b2b2b)
- 加载中: 黄色指示器 (#ffb74d)
- 错误: 红色错误信息 (#f44336)
- 悬停: 亮灰背景 (#343434)

**趋势系统**:
- 上升 (↑): 绿色 (#4caf50)
- 下降 (↓): 红色 (#f44336)
- 平稳 (→): 灰色 (#9e9e9e)

**核心方法**:
```python
set_value(value, trend, subtitle)     # 更新KPI数值
set_loading(is_loading)                # 显示加载状态
set_error(error_msg)                   # 显示错误状态
bind_click(callback)                   # 绑定点击事件
_format_value(value)                   # 格式化显示值
_update_display()                      # 刷新显示
```

### 2. DashboardV2Frame 主框架 (Lines 230-537)

**布局结构**:
- 工具栏 (刷新、自动刷新开关、导出按钮)
- KPI网格 (2行4列, 响应式权重)
- 状态栏 (加载状态 + 最后刷新时间)

**数据加载**:
```
load_data() -> is_loading=True
    ↓
_load_data_async() [线程] -> _collect_dashboard_metrics()
    ↓
ReportEngine methods或 Empty metrics
    ↓
_apply_metrics() -> Update UI cards
```

**8个KPI数据方法**:
1. `_get_monthly_revenue()` - 本月收入
2. `_get_annual_revenue()` - 年度收入
3. `_get_active_contracts()` - 活跃合同数
4. `_get_unpaid_amount()` - 未收总额
5. `_get_overdue_contracts()` - 逾期合同数
6. `_get_payment_rate()` - 收款率
7. `_get_monthly_exchanges()` - 本月换机次数
8. `_get_high_risk_customers()` - 高风险客户数

**高级特性**:
- **方法别名**: `_call_engine_method()` 尝试多个方法名
- **指标规范化**: `_normalize_metrics()` 处理不同命名约定
- **回退机制**: `_empty_metrics()` 无数据源时返回零值
- **副标题生成**: `_build_subtitles()` 为每个KPI添加上下文
- **趋势计算**: `_calculate_trend()` 对比当前vs历史数据

## 🎓 技术亮点

### 架构设计
1. **模块化**: KpiCard独立, DashboardV2Frame容器
2. **关注点分离**: 数据层(ReportEngine) vs UI层(Dashboard)
3. **可扩展性**: 易于添加新KPI或报表类型
4. **容错能力**: 多层回退机制

### 异步处理
1. **线程**: 数据加载不阻塞UI
2. **主线程回调**: `after(0, callback)` 安全更新UI
3. **状态管理**: `is_loading`标志防止并发加载
4. **定时器**: `after(interval, func)` 实现自动刷新

### 数据管理
1. **缓存策略**: 当前数据 + 历史数据
2. **趋势计算**: 对比当前vs历史，得出上升/下降/平稳
3. **格式化**: 数字千位分隔，百分比保留小数
4. **规范化**: 支持多种KPI命名约定

### 错误处理
1. **加载错误**: 显示错误消息，用户可重试
2. **无数据源**: 返回零值，界面仍可用
3. **无效数据**: 类型转换异常捕获，安全处理
4. **用户反馈**: 状态栏实时显示加载状态
