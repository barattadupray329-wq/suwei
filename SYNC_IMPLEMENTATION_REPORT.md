## 🚀 工作流程

### 主电脑（开发机器）
```
启动应用
  ↓
用户登录
  ↓
自动启动 HTTP 服务器 (:9999)
  ↓
显示服务器 IP（例如 192.168.1.100:9999）
  ↓
修改代码（modules/, core/, theme/）
  ↓
UpdateNotifier 自动检测（每 30 秒）
  ↓
生成更新包到 租赁数据\.update_packages\
  ↓
HTTP 服务器提供访问
```

### 其他电脑（客户端）
```
运行配置脚本
  ↓
输入主电脑 IP（例如 192.168.1.100:9999）
  ↓
脚本测试连接 ✓
  ↓
exe 启动时检查更新（每 60 秒）
  ↓
发现新更新
  ↓
自动下载
  ↓
应用更新
  ↓
自动重启应用 ✓
```

## 🌐 API 架构

### 服务器端点（主电脑）

```
GET /api/status                          获取服务器状态
GET /api/updates/list                    获取可用更新列表
GET /api/updates/download/{filename}     下载更新包
GET /api/data/list                       获取数据文件列表
GET /api/data/download/{filepath}        下载数据文件
```

### 配置文件

**主电脑配置**（自动生成）：`sync_server_config.json`
```json
{
  "server_ip": "192.168.1.100",
  "port": 9999,
  "started_at": "2026-06-14T10:00:00",
  "project_root": "/path/to/project",
  "data_folder": "/path/to/data",
  "update_folder": "/path/to/data/.update_packages"
}
```

**客户端配置**（手动创建）：`sync_client_config.json`
```json
{
  "server_url": "http://192.168.1.100:9999",
  "check_interval": 60,
  "auto_update": true,
  "data_sync_enabled": true,
  "log_level": "INFO"
}
```

## 🎓 开发过程

### 设计决策

✅ **为什么选择 HTTP 服务器而不是其他方案？**
- 无需额外软件安装
- 完全可控，代码在项目中
- 局域网速度快
- 跨平台兼容

✅ **为什么不使用线程池？**
- HTTPServer 本身支持并发
- 简化设计，减少复杂度
- 满足当前需求

✅ **为什么配置分为主电脑和客户端？**
- 主电脑自动生成，无需手动配置
- 客户端手动配置一次永久使用
- 灵活应对不同部署场景
